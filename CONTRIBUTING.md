# 贡献指南

感谢你关注本项目。提交改动前，请先确认它不会记录、上传或展示任何真实课表、学生信息、登录 Cookie 或其他个人数据。

## 本地开发

- 修改 `userscripts/xdf-schedule-export.user.js` 后，先用 `node --check` 验证语法。
- 首次调试时，按 [油猴脚本说明的本地联调步骤](userscripts/README.md#本地联调与断点调试) 配置一个仅供开发的 Tampermonkey 加载器。之后在 VS Code 保存源码、刷新测试页面即可生效，无需反复复制粘贴。
- 在有权访问的测试账号中验证日期选择、导出顺序与失败重试。开发加载器与正式安装脚本不要同时启用。

## 提交建议

- 保持脚本在当前稳定版 Tampermonkey 与 Chromium 浏览器中可用。
- 为行为改动补充或更新验证步骤。
- 不要提交真实导出的 Excel、接口响应、截图或任何包含个人信息的文件。
- 如果涉及课表接口，请只在你有权访问的账号和环境中验证，并保持低频请求。

## 报告安全问题

请不要在公开 Issue 中粘贴 Cookie、账号信息、接口响应或真实课表。若发现可能泄露数据的问题，请联系仓库维护者并提供已脱敏的复现步骤。
